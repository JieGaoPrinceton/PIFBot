## 2. 数据流分析

### 2.1 传感器数据流（上行）

```
传感器 → 控制器/SBC → ROS Topics → SLAM/导航算法
```

| 传感器 | 数据类型 | 发布频率 | 作用 |
|-------|---------|---------|------|
| RPLIDAR A1 | LaserScan | ~5.5 Hz | 2D 环境扫描 |
| RealSense D415 | PointCloud2 + Image | 30 Hz | 3D 深度信息 |
| 编码器 | Odometry | ~50 Hz | 里程估计 |
| MPU9250 | Imu | ~100 Hz | 姿态融合 |

### 2.2 控制命令流（下行）

```
导航算法 → /cmd_vel Topic → rosserial → 控制器 → 电机
```

## 4. 通信协议

### 4.1 rosserial 协议

控制器板与 ROS 系统之间使用 rosserial 协议通信：

| 参数 | 值 |
|-----|-----|
| 传输层 | UART (Serial) |
| 波特率 | 115200 bps (典型值) |
| 数据格式 | JSON/二进制封装 |

### 4.2 消息类型

```yaml
# 速度命令
geometry_msgs/Twist:
  linear:
    x: 前进速度 (m/s)
    y: 0 (差速驱动不支持)
    z: 0
  angular:
    x: 0
    y: 0
    z: 角速度 (rad/s)

# 里程计
nav_msgs/Odometry:
  header:
    frame_id: "odom"
  child_frame_id: "base_link"
  pose:
    position: {x, y, z}
    orientation: {x, y, z, w}
  twist:
    linear: {x, y, z}
    angular: {x, y, z}
```

## 6. 系统工作流程

### 6.1 SLAM 建图流程

1. 激光雷达扫描环境 → 发布 `/scan`
2. 控制器计算里程 → 发布 `/odom`
3. TF 变换更新
4. SLAM 算法处理数据 → 发布 `/map`
5. 用户通过 RViz 监控建图过程

### 6.2 导航流程

1. 加载已构建地图
2. 设置目标点
3. 全局路径规划
4. 局部路径规划（避障）
5. 发布速度命令 `/cmd_vel`
6. 控制器执行运动
7. 持续反馈和调整
