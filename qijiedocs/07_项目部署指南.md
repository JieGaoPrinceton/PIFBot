# PIFBot 项目部署指南

## 1. 环境准备

### 1.1 系统要求

| 组件 | 要求 |
|-----|------|
| 操作系统 | Ubuntu 16.04 LTS (Xenial) |
| ROS 版本 | ROS Kinetic Kame |
| Python | 2.7 (ROS Kinetic 默认) |

### 1.2 安装 ROS Kinetic

```bash
# 添加 ROS 软件源
sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'

# 添加密钥
sudo apt-key adv --keyserver 'hkp://keyserver.ubuntu.com:80' --recv-key C1CF6E31E6BADE8868B172B4F42ED6FBAB17C654

# 更新软件包列表
sudo apt update

# 安装 ROS Kinetic 完整版
sudo apt install ros-kinetic-desktop-full

# 初始化 rosdep
sudo rosdep init
rosdep update

# 添加环境变量
echo "source /opt/ros/kinetic/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## 2. 项目克隆与构建

### 2.1 克隆项目

```bash
# 克隆主仓库
git clone https://github.com/phonght32/PIFBot.git
cd PIFBot

# 初始化并更新所有子模块
git submodule update --init --recursive
```

### 2.2 ROS 工作空间构建

```bash
# 进入 ROS 目录
cd ros

# 安装依赖
rosdep install --from-paths src --ignore-src -r -y

# 编译工作空间
catkin_make

# 或者使用 catkin build (需安装 catkin-tools)
# sudo apt install python-catkin-tools
# catkin build

# 激活工作空间
source devel/setup.bash
```

## 3. 硬件组装

### 3.1 材料清单 (BOM)

| 组件 | 数量 | 说明 |
|-----|------|------|
| OpenSTM32 控制板 | 1 | 主控制器 |
| Raspberry Pi 3 B+ | 1 | 单板计算机 |
| RPLIDAR A1 | 1 | 激光雷达 |
| Intel RealSense D415 | 1 | 深度相机 |
| 42×42mm 步进电机 | 2 | 驱动电机 |
| TB6560 驱动器 | 2 | 电机驱动 |
| 65mm 轮子 (带编码器) | 2 | 驱动轮 |
| 万向轮 | 2 | 支撑轮 |
| MPU9250 | 1 | IMU (板载) |
| 3S LiPo 电池 | 1 | 11.1V, ≧3000mAh |
| LM2596 DC-DC 模块 | 1 | 降压模块 |
| 铜柱 M3 | 若干 | 支撑结构 |
| 螺丝螺母 M3 | 若干 | 紧固件 |

### 3.2 机械加工

```bash
# 使用 DXF 文件进行激光切割
# 文件位置: mechanical/assets/dxf/

layer1.dxf  → 底层板 (控制层)
layer2.dxf  → 中层板 (计算层)
layer3.dxf  → 顶层板 (传感器层)
camera_holder.dxf → 相机支架
```

### 3.3 电子连接

```
接线顺序:
1. 电机驱动接线
   - 步进电机 → TB6560 输出端
   - TB6560 控制端 → OpenSTM32 GPIO

2. 编码器接线
   - 编码器 A/B 相 → OpenSTM32 定时器

3. IMU 接线
   - MPU9250 I2C → OpenSTM32 I2C1

4. 通信接线
   - OpenSTM32 UART → Raspberry Pi GPIO UART/USB

5. 电源接线
   - 电池 → LM2596 输入
   - LM2596 输出 → Raspberry Pi/OpenSTM32 5V
   - 电池直接 → TB6560 12V
```

## 4. 固件烧录

### 4.1 开发环境设置

```bash
# 安装 ARM 工具链
sudo apt install gcc-arm-none-eabi

# 安装 OpenOCD (调试)
sudo apt install openocd

# 或使用 STM32CubeIDE (图形化 IDE)
# 下载: https://www.st.com/en/development-tools/stm32cubeide.html
```

### 4.2 编译与烧录

```bash
# 进入固件目录
cd firmware/ros-stm32-base-control

# 编译 (根据项目 Makefile)
make

# 烧录 (使用 ST-Link 或 OpenOCD)
make flash
# 或
openocd -f interface/stlink.cfg -f target/stm32f4x.cfg -c "program build/firmware.elf verify reset exit"
```

## 6. 运行系统

### 6.1 启动最小系统

```bash
# 终端 1: 启动 ROS Master
roscore

# 终端 2: 启动 rosserial
rosrun rosserial_python serial_node.py /dev/ttyUSB0

# 终端 3: 检查话题
rostopic list
rostopic echo /odom
```

### 6.2 启动 SLAM 建图

```bash
# 使用 launch 文件启动 (如果已配置)
roslaunch pifbot_bringup slam.launch

# 或手动启动各节点
# 终端 1: roscore
# 终端 2: rosserial
# 终端 3: RPLIDAR
rosrun rplidar_ros rplidarNode
# 终端 4: GMapping
rosrun gmapping slam_gmapping scan:=/scan
# 终端 5: RViz
rosrun rviz rviz
```

### 6.3 启动导航

```bash
# 启动导航
roslaunch pifbot_bringup navigation.launch map:=/path/to/map.yaml

# 或手动启动
# 1. 地图服务器
rosrun map_server map_server mymap.yaml
# 2. AMCL
rosrun amcl amcl
# 3. Move Base
rosrun move_base move_base
```

## 8. 调试与故障排除

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| rosserial 无法连接 | 串口权限/波特率不匹配 | 检查权限、确认波特率 |
| 无里程计数据 | 编码器接线/固件问题 | 检查接线、调试固件 |
| SLAM 地图质量差 | 激光雷达校准/里程计漂移 | 校准传感器、检查 TF |
| 导航不走 | Costmap 配置/目标设置 | 检查配置、确认目标可达 |

### 8.2 调试命令

```bash
# 检查节点
rosnode list
rosnode info /node_name

# 检查话题
rostopic list
rostopic echo /topic_name
rostopic hz /topic_name

# 检查 TF
rosrun tf view_frames
evince frames.pdf

# 检查服务
rosservice list
rosservice call /service_name

# 可视化
rqt_graph
rqt_tf_tree
```

## 9. 性能优化

### 9.1 参数调优

| 参数 | 建议范围 | 说明 |
|-----|---------|------|
| PID Kp | 0.5 - 2.0 | 比例增益 |
| PID Ki | 0.0 - 0.5 | 积分增益 |
| PID Kd | 0.0 - 0.2 | 微分增益 |
| 地图分辨率 | 0.03 - 0.05 m | 越小越精细 |
| 控制频率 | 10 - 20 Hz | Move Base |

### 9.2 计算优化

```yaml
# 降低传感器分辨率以减少计算负担
RealSense:
  depth_resolution: 640x480 (原 1920x1080)
  fps: 15 (原 30)

RPLIDAR:
  模式: 标准 (非增强)

SLAM:
  particles: 20-30 (GMapping)
```
